# Future of Software Engineering

How AI agents are transforming the engineering profession. A consistent theme across sources — not "are jobs going away?" but "how is the role changing?"

## The Karpathy Thesis: Jevons Paradox

[[Andrej Karpathy]] applies **Jevons Paradox** to coding: as AI makes coding cheaper, total demand for code explodes rather than shrinks. Historical parallel: when steam engines became more efficient, total coal consumption increased because new use cases opened up.

Implication: **coding jobs aren't going anywhere — they're multiplying in new forms.**

## The Role Shift

Sources converge on a transformation from "writing code" to "managing agents":

- **Before**: engineer writes code, reviews code, debugs code
- **After**: engineer defines specs, orchestrates agents, verifies output, curates quality

From Stanford/Mihail Eric: managing multiple agents well is described as **"the last boss"** — only the top 1% of users truly excel at it. The skill is closer to team management than programming.

Birgitta Boeckeler: the push is toward **more autonomy, less supervision** — cloud agents running unsupervised with context engineering as the key skill.

## The Five Meta-Skills

From multiple sources, the new engineering skillset:

1. **Spec writing** — clear, unambiguous requirements (see [[Prompting for Claude]])
2. **[[Context Management|Context engineering]]** — managing what the AI sees and when
3. **Harness engineering** — mastering your tools deeply ([[Claude Code]], hooks, skills, MCP)
4. **Agent orchestration** — coordinating multiple AI agents on complex tasks
5. **Verification** — building systems that catch AI errors (see [[Verification-Driven Development]])

## The 5-Year Plan is Dead

One source argues the career planning horizon has collapsed from 5 years to 5 months. Engineers who don't adapt to agent-assisted workflows risk obsolescence not in years but in quarters.

## Phase Transition (December 2025)

Multiple sources mark December 2025 as a phase transition:
- Three frontier models (Claude, Gemini, GPT) launched within six days
- All optimized for sustained autonomous operation
- [[Anthropic]]'s engineers reportedly stopped writing code themselves — using AI agents for most development
- [[OpenAI]] slowing hiring as agents replace some engineering roles

## Contrarian View: Coding Isn't Dead

[[Andrej Karpathy]] pushes back hardest: engineering fundamentals still matter. You need to understand what you're asking agents to build. The agents amplify skill — they don't replace it. A great engineer with agents is 100x more productive. A non-engineer with agents produces 100x more bugs.
